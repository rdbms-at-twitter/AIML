# RAG チャンク処理フロー

## データ取り込み（Sync 時に実行）

```
S3 上の PDF
    │
    ▼
┌─────────────┐
│ パーサー     │  PDF → テキスト抽出
└─────────────┘
    │
    ▼
┌─────────────────────────────┐
│ チャンク分割                 │
│ ・固定サイズ (例: 300トークン) │
│ ・オーバーラップあり (例: 20%) │
│ → 1つのPDFが数百〜数千チャンクに │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 埋め込みモデル               │
│ (Titan Embeddings 等)        │
│ 各チャンク → 1024次元ベクトル │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ OpenSearch Serverless に保存  │
│ ・ベクトル                   │
│ ・元テキスト                 │
│ ・メタデータ (ファイル名等)   │
└─────────────────────────────┘
```

## クエリ時（Retrieve API 呼び出し時）

```
質問テキスト
    │
    ▼
┌─────────────────────────────┐
│ 同じ埋め込みモデルでベクトル化 │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ OpenSearch で kNN 検索        │
│ → コサイン類似度で上位N件取得 │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ スコア付きチャンク一覧を返却  │
│ (score: 0.541, text: "...")  │
└─────────────────────────────┘
```

## チャンク設定（KB の Data Source で変更可能）

| 設定 | 説明 |
|------|------|
| チャンクサイズ | 大きい → 文脈豊富だが検索精度低下、小さい → 精度高いが文脈不足 |
| オーバーラップ | チャンク間の重複。情報の切れ目での欠損を防ぐ |
